LAB REPORT: VOIP NETWORK CONFIGURATION

Implementation of Voice Services in a Cisco Environment

Course: Computer Networking & Systems
Topic: Router & Switch VoIP Setup
Submitted By: Md Ashikur Rahman
Date: Aug 24, 2025

1. Objective

The primary objective of this lab is to configure a functional Voice over IP (VoIP) network using a Cisco Router and Switch. This includes setting up a Voice VLAN, DHCP services for IP phones, and configuring Telephony Services to assign directory numbers.

2. Network Topology & Components

  • Router: Cisco 2811 (Acting as a Call Manager Express)
  • Switch: Cisco 2960 (Layer 2 Switching with Voice VLAN support)
  • End Devices: 4x Cisco 7960 IP Phones
  • Cabling: Copper Straight-Through

3. Configuration Steps

Step 1: Switch Configuration (Voice VLAN)

Define a specific VLAN to prioritize voice traffic and ensure quality of service (QoS).

Switch> enable Switch# configure terminal Switch(config)# vlan 10 Switch(config-vlan)# name VOICE_VLAN Switch(config-vlan)# exit Switch(config)# interface range fa0/1-6 Switch(config-if-range)# switchport mode access Switch(config-if-range)# switchport voice vlan 10

Step 2: Router Sub-interface & DHCP

Configure the router to handle Inter-VLAN routing and provide IP addresses automatically to the phones.

Router(config)# interface fa0/0.10 Router(config-subif)# encapsulation dot1Q 10 Router(config-subif)# ip address 192.168.10.1 255.255.255.0 Router(config)# ip dhcp pool VOICE_DHCP Router(dhcp-config)# network 192.168.10.0 255.255.255.0 Router(dhcp-config)# default-router 192.168.10.1 Router(dhcp-config)# option 150 ip 192.168.10.1

Step 3: Telephony Service Setup

Enable Call Manager functions on the router to manage the IP phones.

Router(config)# telephony-service Router(config-telephony)# max-dn 5 Router(config-telephony)# max-ephones 5 Router(config-telephony)# ip source-address 192.168.10.1 port 2000 Router(config-telephony)# auto assign 1 to 5

Step 4: Directory Number Assignment

Assigning unique extension numbers (DN) to each connected phone.

Router(config)# ephone-dn 1 Router(config-ephone-dn)# number 101 Router(config)# ephone-dn 2 Router(config-ephone-dn)# number 102 Router(config)# ephone-dn 3 Router(config-ephone-dn)# number 103

4. Conclusion

After completing the configuration, all IP phones successfully requested and received IP addresses from the DHCP pool. The Telephony Service correctly identified the devices and assigned the predefined directory numbers. Test calls between extensions 101, 102, and 103 were successful, confirming the network's readiness for voice communication.
